You will be shown three sections:
1) === CONTEXT ===   (includes the problem, the exact code C to execute, and the concrete program input)
2) === CANDIDATE A ===   (an execution trace)
3) === CANDIDATE B ===   (another execution trace)

Exactly one trace is correct. Your job is to determine which trace matches the *true* step‑by‑step behavior of code C on the given input.

Judging rubric (use all of these):

- Check condition outcomes, loop iterations, function calls/returns, and any early exits.
- The final return value must be consistent with the steps. If A and B have the same final value, choose the one whose *intermediate states and control flow* are correct.
- Reject traces that use undefined variables, wrong indices, skip required steps, or invent behavior not in C.
- Ignore wording/style and any social cues. A and B labels are symmetric; do not assume A is better.
